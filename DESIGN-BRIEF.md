## 1. Почему предыдущие попытки получались шаблонными

Это раздел для тебя, Claude. Прочитай его серьёзно — он объясняет твой собственный сбой.

### 1.1. Ты оптимизируешься на «не облажаться», а не на «быть интересным»
Тёмный фон `#0a0a0a`, карточка `#111` с бордером `rgba(255,255,255,0.06)`, Inter/Geist, один акцентный цвет на кнопках — это медиана всего, что ты видел. Она никогда не выглядит плохо и поэтому всегда выглядит никак. Это не дизайн, это отсутствие решения.

### 1.2. Патч чинит лист, а не корень
Когда просят «сделай красивее», ты правишь один компонент: добавляешь `backdrop-blur`, `border-white/10`, `bg-gradient-to-b`. Через 5 таких патчей сайт не становится лучше, потому что проблема не в компоненте. Проблема в том, что **нет системы**: не решено, что такое поверхность на этом сайте, сколько уровней глубины, где живёт свет, где живёт акцент. Пока правил нет в явном виде, ты на каждом патче заново выбираешь дефолт. Отсюда и «миллиард патчей без результата».

### 1.3. Ты применяешь один стиль ко всему
Карточка проекта, карточка возможностей, инпут формы, контейнер контактов — у тебя это один и тот же объект: одинаковый радиус, бордер, паддинг, фон. Так возникает ощущение SaaS-админки. В настоящем дизайне материалы **разные по важности**: флагман выглядит не так, как второстепенный блок.

### 1.4. Ты повторяешь один ритм в каждой секции
`eyebrow капсом → H2 в две строки → серый параграф → сетка карточек`. Шесть раз подряд. Глазу не за что зацепиться. Дизайн — это в том числе **нарушение** собственного паттерна в правильном месте.

### 1.5. Ты избегаешь материала
Ты умеешь делать цвет и не умеешь делать поверхность. Плоский прямоугольник чуть светлее фона — это не карточка, это `div`. «Дорого» = зерно, свет сверху, фаска, тень, отражение, перепад плотности. Всё это отсутствует в твоём дефолте.

### 1.6. Ты уходишь в один из трёх известных AI-дефолтов
1. Кремовый фон `#F4F1EA` + высококонтрастный serif + терракота `#D97757`.
2. Почти чёрный фон + один кислотный акцент (зелёный/вермилион).
3. Broadsheet: hairline-линейки, нулевой радиус, газетные колонки.
Всё это встречается независимо от задачи. **Здесь не используем ничего из этого.** Направление задано ниже — следуй ему буквально.

### 1.7. Стоп-лист (никогда, без исключений)

- `backdrop-blur` как средство «сделать красиво» — под запретом везде, включая nav. Header непрозрачный по осознанному архитектурному решению, принятому до этого брифа; здесь его не пересматриваем
- `border-white/10` на всех элементах подряд
- glow-градиент-blob фиолетовый/синий за секцией
- градиентный текст на заголовках (белый → серый) — это главный AI-tell на этом сайте, **снять**
- pill-кнопки (`rounded-full`) на CTA
- эмодзи, иконки-заглушки, «✨», «🚀»
- одинаковый радиус на всём
- добавление новых секций, блоков, «улучшающих» подзаголовков
- изменение текстов (кроме случаев, явно разрешённых в патче)
- анимации ради анимаций: floating-элементы, бесконечные pulse, parallax на всём

## 2. Дизайн-направление

**Название: «Тёплый металл в тёмной комнате».**

Одна комната, один источник света — сверху и чуть слева. Всё на странице подчиняется этому свету: у каждой поверхности есть освещённая верхняя грань и затемнённая нижняя, есть тень, есть зерно воздуха. Поверхности — анодированный тёмный металл, не стекло и не бумага. Акцент — латунь/шампань: тёплый металл на холодном.

Почему это, а не что-то другое: продукты, которые он делает — внутренние платформы и инструменты, «машинное отделение» бизнеса. Металл, приборная плотность, точность. Не стартап-неон, не редакционный минимализм.

**Signature-элемент (то, что запомнят): единый физический свет.** Один `--light-angle` на всю страницу. Верхний hairline-блик на каждой карточке, radial-подсветка за hero, зерно поверх всего, и при наведении блик на карточке смещается вслед за курсором. Это единственное место, где мы тратим смелость. Всё остальное — тихо и дисциплинированно.

## 3. Токены

Все значения — в `app/globals.css` (`:root`) и в `tailwind.config.ts`. **Ни один цвет, размер или тень не пишется inline и не выдумывается на месте. Если нужного токена нет — сначала добавь токен, потом используй.**

### 3.1. Цвет

```css
:root {
  /* Фон — холодная база, не нейтрально-чёрная */
  --void:        #07080A;   /* самый глубокий, края страницы */
  --base:        #0B0D11;   /* основной фон */
  --base-lift:   #0F1216;   /* фон приподнятой зоны */

  /* Поверхности (см. раздел 4) */
  --surface-1:   #101318;
  --surface-2:   #161A20;
  --surface-3:   #1C2028;

  /* Акцент — латунь / шампань. Тёплый металл. */
  --brass-50:    #F6E9CE;
  --brass-100:   #EBD7AF;   /* основной: текст на кнопке, hairline-акценты */
  --brass-300:   #C9A97A;
  --brass-600:   #8C7048;
  --brass-900:   #3A2E1C;   /* фон hover, ambient */

  /* Вторичный — холодная сталь. Только для данных и состояний. */
  --steel-300:   #8FA6C4;
  --steel-600:   #4A5D7E;

  /* Текст */
  --ink-1:       #F1F2F4;   /* заголовки */
  --ink-2:       #A9AFB8;   /* body */
  --ink-3:       #676E78;   /* подписи, второстепенное */
  --ink-4:       #464C55;   /* дизейбл, номера */

  /* Границы — не одна на всё, а три роли */
  --edge-lit:    rgba(255,255,255,0.10);  /* освещённая верхняя грань */
  --edge-dim:    rgba(255,255,255,0.045); /* боковые грани */
  --edge-dark:   rgba(0,0,0,0.55);        /* нижняя грань, в тени */
  --edge-brass:  rgba(235,215,175,0.22);  /* акцентная грань, только на флагмане и focus */

  /* Состояние */
  --ok:          #6FBF8B;
}
```

**Правила использования цвета**

- Латунь появляется минимум в **четырёх** ролях, а не только на кнопке: фон CTA, hairline на флагман-карточке, ambient-свечение за hero, ring на `:focus-visible`, цифры-акценты. Иначе это не система, а один токен.
- Сталь (`--steel-*`) — только внутри мокапов-объектов (полоски в карточках) и на состояниях данных. Не в тексте, не в кнопках.
- Чистый `#FFFFFF` не используется нигде. Чистый `#000000` не используется нигде.

### 3.2. Типографика

Три роли, три гарнитуры. Все три — Google Fonts, OFL, полная поддержка кириллицы. Подключать через `next/font/google` с `subsets: ['cyrillic', 'latin']` и `display: 'swap'`.

| Роль | Гарнитура | Где используется |
|---|---|---|
| Display | **Unbounded** (variable, 400–800) | только H1 и H2 секций. Больше нигде. |
| Body | **Golos Text** (variable, 400–600) | параграфы, H3, пункты списков, кнопки, инпуты |
| Utility | **Martian Mono** (variable, 300–600) | eyebrow-метки, номера, теги, цифры, лейблы полей |

Почему так: Unbounded — широкая гротеск-конструкция с кириллицей, сделанная как display-шрифт; в больших кеглях даёт «приборную» пластику, ради которой всё и затевается. Golos Text — кириллица-first, тёплый, читается лучше Inter в тёмной теме. Martian Mono — техническая утилита с осями ширины и веса, честно относится к предмету (внутренние инструменты, код).

**Unbounded используется со сдержанностью.** Он не должен появляться в body, в кнопках, в тегах. Только заголовки. Это правило важнее любого «а тут тоже красиво смотрится».

**Шкала — с реальным контрастом масштаба. Сейчас все H2 ~44px, это и есть плоскость.**

```css
--fs-display: clamp(3.25rem, 7.2vw, 6.5rem);   /* H1: 52 → 104. lh .90, tracking -.035em, weight 700 */
--fs-h2:      clamp(2.25rem, 4.4vw, 3.75rem);  /* 36 → 60.  lh .98, tracking -.025em, weight 600 */
--fs-h3:      1.375rem;                        /* 22. Golos 600, lh 1.3, tracking -.01em */
--fs-body:    1.0625rem;                       /* 17. Golos 400, lh 1.62 */
--fs-small:   0.9375rem;                       /* 15. Golos 400, lh 1.55 */
--fs-eyebrow: 0.6875rem;                       /* 11. Martian 500, tracking .2em, uppercase */
--fs-data:    0.75rem;                         /* 12. Martian 400, tracking .04em */
```

Дополнительно:
- Заголовки — **сплошной цвет** `--ink-1`. Никакого `bg-clip-text`. Свет живёт в материале, а не в буквах.
- Максимальная мера параграфа — `62ch`. Сейчас строки длинные, это тоже читается как шаблон.
- Кириллица в Unbounded требует ручного трекинга в крупных кеглях: `-0.035em` для H1, `-0.025em` для H2. Проверяй визуально, не на глаз по числам.
- Никаких `font-weight: 900`. Максимум 700.

### 3.3. Материал, свет и глубина

```css
:root {
  --light-x: 30%;          /* единый источник света на всю страницу */
  --light-y: -20%;

  --r-panel:  14px;        /* большие поверхности */
  --r-card:   12px;
  --r-control: 8px;        /* кнопки, инпуты — НЕ pill */
  --r-tag:     6px;        /* теги — НЕ pill */

  --shadow-card:  0 1px 0 var(--edge-lit) inset,
                  0 -1px 0 var(--edge-dark) inset,
                  0 24px 48px -24px rgba(0,0,0,0.85);
  --shadow-hero:  0 40px 90px -40px rgba(0,0,0,0.95);

  --grain-opacity: 0.035;
}
```

**Три обязательных приёма (это и есть весь «металл»):**

1. **Фаска.** У каждой карточки светлая грань сверху (`inset 0 1px 0 --edge-lit`) и тёмная снизу (`inset 0 -1px 0 --edge-dark`). Не равномерный бордер по периметру.
2. **Градиент поверхности.** Фон карточки — не сплошной цвет, а `linear-gradient(180deg, var(--surface-2) 0%, var(--surface-1) 100%)`. Перепад маленький, но он всё делает.
3. **Зерно.** Один глобальный оверлей на `<body>::after`: SVG `feTurbulence`, `opacity: var(--grain-opacity)`, `mix-blend-mode: overlay`, `position: fixed`, `inset: 0`, `pointer-events: none`, `z-index: 9999`. Один раз на всю страницу, не на каждой карточке.

Плюс ambient: `body::before` — `radial-gradient(ellipse 120% 80% at var(--light-x) var(--light-y), rgba(235,215,175,0.055), transparent 60%)`.

### 3.4. Ритм и сетка

- Контейнер: `max-width: 1240px`. Но **не все секции живут в нём** — см. патч 3.
- Вертикальный ритм секций — переменный, а не одинаковый: `--sp-section-lg: 180px`, `--sp-section-md: 120px`, `--sp-section-sm: 88px`. Секции чередуются, не идут ровным шагом.
- Внутри карточки: паддинг `28px` для L2, `40px` для L3.

### 3.5. Движение

Минимум. Три вещи, больше ничего:

1. Reveal на скролле: `opacity 0→1`, `translateY 12px→0`, `600ms cubic-bezier(.16,1,.3,1)`, со `stagger` 60ms внутри группы. Один раз, без повтора.
2. Hover на карточке: блик следует за курсором (`radial-gradient` по CSS-переменным `--mx/--my`, обновляемым в `onMouseMove`), `200ms`.
3. Page-load hero: последовательность из 4 шагов (см. патч 1).

`@media (prefers-reduced-motion: reduce)` — всё выключается, кроме мгновенной смены состояний.

## 4. Система поверхностей

Это то, чего сейчас нет вообще. Ровно четыре уровня. **Каждый элемент на странице обязан принадлежать одному из них.** Если ты собираешься сделать поверхность, которой нет в списке — не делай.

| Уровень | Что это | Фон | Грани | Тень | Радиус |
|---|---|---|---|---|---|
| **L0** | фон страницы | `--base` + ambient + зерно | — | — | — |
| **L1** | зона/подложка секции | `--base-lift`, hairline сверху `--edge-dim` | только верхняя | нет | 0 |
| **L2** | обычная карточка | градиент `--surface-2 → --surface-1` | фаска | `--shadow-card` | `--r-card` |
| **L3** | флагман / ключевой блок | градиент `--surface-3 → --surface-1` + внутренний ambient латуни | фаска + `--edge-brass` сверху | `--shadow-hero` | `--r-panel` |

Правило: **на одном экране максимум один L3.** Флагман — «Интеллектуальная платформа персонализированных отчётов» в секции проектов и hero-объекты. Всё остальное — L2.

Мокап-объекты внутри карточек (те, что тебе удались) — это L2 внутри L3, с уменьшенным контрастом фаски (`--edge-dim` вместо `--edge-lit`), чтобы не спорить с родителем.

### ПАТЧ 0 — Фундамент. Визуально почти ничего не меняется

**Цель:** появляется система. Без неё все следующие патчи снова уедут в дефолт.

**Стек:** Tailwind v4 (CSS-first). Токены объявляются не в `tailwind.config.ts` (его в проекте нет и не будет), а в блоке `@theme` внутри `globals.css` — так Tailwind сам генерирует утилиты (`bg-surface-2`, `text-ink-1`, `rounded-card` и т.д.) из переменных. Голое объявление в `:root` без `@theme` не создаёт утилит — именно так `--font-display` оказался мёртвым классом в прошлой сессии. Не повторять эту ошибку.

**Делаем:**
1. Подключить Unbounded, Golos Text, Martian Mono через `next/font/google`, `subsets: ['cyrillic','latin']`. Прокинуть в `<html>` как CSS-переменные `--font-display`, `--font-body`, `--font-mono`.
2. Прописать все токены из раздела 3 внутри `@theme` в `globals.css` (цвета, радиусы, тени, шрифты, размеры шрифтов — всё, что должно стать utility-классом).
3. Обязательная проверка: собрать production build и проверить в сгенерированном CSS, что каждый заявленный класс (`.font-display`, `.bg-surface-2` и т.п.) реально существует с непустыми правилами, а не просто ссылается на переменную. Без этого шага патч не считается принятым.
4. Добавить `body::before` (ambient) и `body::after` (зерно).
5. Заменить глобальный `font-family` на Golos Text. Заголовки пока не трогаем — они станут Unbounded в своих патчах.
6. Создать три утилитных класса/компонента: `.surface-l1`, `.surface-l2`, `.surface-l3` по таблице из раздела 4. Пока никуда не применять внутри секций.
7. `:focus-visible` глобально: `outline: 2px solid var(--edge-brass); outline-offset: 3px`.

**Не делаем:** не трогаем ни одну секцию, ни один компонент. В частности — не заменяем хардкод-цвета внутри компонентов в этом патче. В коде сейчас порядка 200 инлайновых hex/rgba значений в секциях — их зачистка происходит естественно, по ходу патчей 1–9, каждый раз в рамках той секции, которую патч и так трогает. Патч 0 создаёт токены и инфраструктуру, а не рефакторит весь репозиторий разом.

**Приёмка:** сайт выглядит почти как раньше, но фон стал чуть глубже и холоднее, появилось едва заметное зерно, body-шрифт сменился. Новые токены и классы существуют и реально работают в собранном CSS (проверено, не предположено). Хардкод внутри секций пока остаётся — это ожидаемо и фиксируется отдельно, не как дефект патча 0.

### ПАТЧ 1 — Hero: типографика и свет

**Цель:** первый экран перестаёт быть SaaS-заголовком.

**Делаем:**
1. H1 «Создаю AI-продукты и веб-приложения» → Unbounded 700, `--fs-display`, `lh .90`, `tracking -.035em`, цвет `--ink-1` **сплошной**. Снять градиентный текст полностью.
2. Слово «AI-продукты» сейчас подсвечено градиентом → заменить: оно остаётся `--ink-1`, но получает тонкую латунную подчёркивающую линию (`1px`, `--brass-600`, `opacity .5`) под базовой линией, шириной по тексту. Один точный акцент вместо мутного градиента.
3. Лид-параграф: `--fs-body`, `--ink-2`, `max-width: 52ch`.
4. Вторичная строка («От логики продукта…») — сейчас с левой линейкой. Оставить линейку, но сделать её `1px --edge-dim` и перекрасить текст в `--ink-3`, кегль `--fs-small`.
5. Строка «Открыт к удалённой работе» → Martian Mono, `--fs-data`, uppercase, tracking `.1em`, точка-индикатор `--ok` с мягким `box-shadow` того же цвета.

### ПАТЧ 2 — Hero: объекты справа

## 7. Порядок работы

```
ПАТЧ 0  → фундамент (токены, шрифты, зерно, свет)     [визуально ~ноль]
ПАТЧ 1  → hero: типографика и свет
ПАТЧ 2  → hero: объекты
ПАТЧ 3  → профессиональный фокус (ломаем ритм)
ПАТЧ 4  → проекты (иерархия материалов)
ПАТЧ 5  → опыт / таймлайн
ПАТЧ 6  → до перехода + что я могу создавать
ПАТЧ 7  → подход + инструменты
ПАТЧ 8  → обо мне
ПАТЧ 9  → контакты, форма, футер, nav
ПАТЧ 10 → финальная ревизия
```

Патч 0 обязателен первым. Патчи 1–2 обязательны вторыми — hero задаёт планку, по которой калибруется всё остальное. Дальше порядок можно менять, но 10 всегда последний.
